# Спецификация плагина «UX Content» + промпт для Figma-агента

> Что строим: Figma-плагин, который **генерирует** и **аудирует** интерфейсные тексты по нашим
> правилам (`content-guide.md` + `date-time-formats.md` + глоссарий). Этот файл — спека для сборки
> и готовый промпт в конце (раздел 10), который скармливается Figma-агенту/Make.

---

## 1. Цель
Единообразие интерфейсных текстов. Дизайнер внутри Figma:
- **GENERATE** — получает грамотный вариант текста для выделенного элемента.
- **AUDIT** — прогоняет фрейм/выделение и видит нарушения правил + подозрительные места.

## 2. Источник правил (контекст для LLM)
Плагин держит правила как **системный контекст**:
- `content-guide.md` — ядро (правила `TXT/LNG/TON/STL/TYP/NUM/BTN/ERR/EMP/FRM/HDR/NOT/DLG/GLS/LOC/FMT`).
- `date-time-formats.md` — даты/время (`DT-*`).
- **Глоссарий** — JSON из внешней таблицы (`⚙ GLOSSARY_SOURCE`), схема строки
  `{canonical, synonyms[], note, status, owner, updated_at}`; используем только `status:approved`.

**Как доставлять:** бандлить оба `.md` в код плагина как строковые константы (обновляются при
правке правил — см. правило синка). Глоссарий — тянуть по сети и кэшировать (`figma.clientStorage`).

## 3. Runtime / LLM
- Плагин вызывает LLM-API. **Рекомендация:** Claude (быстрый/дешёвый — Haiku для AUDIT-объёмов,
  Sonnet для генерации).
- Ключ: хранить в `figma.clientStorage` (настройки плагина) ИЛИ ходить через корпоративный
  бэкенд-прокси (безопаснее для ключа). ⚙ РЕШИТЬ (см. §9).
- Температура низкая (0–0.3): нам нужна дисциплина, не креатив.
- Вывод — **строгий JSON** по схемам ниже (см. §6). Парсим, не полагаемся на текст.

## 3a. Дополнительный ввод дизайнера (оба режима)
- **Поле «Ваш контекст / промпт»** (свободный текст) — есть и в GENERATE, и в AUDIT. Добавляется к запросу
  к LLM. Примеры: «сделай дружелюбнее», «упомяни скидку», «это онбординг, тон мягче», «бренды не трогай».
- **Прикрепить изображение** (опц.) — дизайнер может приложить картинку-контекст: скрин задачи из Jira,
  спека, референс. Идёт в мультимодальный запрос. ⚠ Требует мультимодальную модель (Claude vision);
  картинки дороже по токенам → прикладываются по желанию.
- **Авто-скриншот выделения** (опц., nice-to-have) — плагин может сам отрендерить выделение в PNG и
  приложить для визуального контекста. Включаемо флагом.

## 4. Режим GENERATE
**Вход (§9.3 гайда):**
| Поле | Источник | Обяз. |
|---|---|---|
| `element_type` | авто из выделенной ноды (кнопка/ошибка/заголовок/поле/…) | да |
| `current_text` + соседние лейблы | авто из фрейма | — |
| `moment` (`рутина/успех/ошибка/онбординг/оплата-безопасность`) | дропдаун, предзаполнен догадкой | да |
| `intent` — «о чём экран, что хотим сказать» | поле дизайнера (наследуется с флоу/экрана) | да |
| `constraints` (длина, тон-override) | опционально | — |

**UX:** две кнопки — **«Сгенерировать»** (с нуля) и **«Улучшить»** (мой текст). Работаем по
**выделению**: элемент / секция / фрейм / экран / компонент. Ничего не выделено — подсказка
«Выберите элемент». Если в выделении несколько текстов (секция/экран) — обрабатываем все.

**Логика:** system = правила; user = вход выше. LLM возвращает **1–3 варианта** + какие правила
держатся. Не выдумывать факты о продукте → плейсхолдер `{{...}}`. Термин — из глоссария (`approved`);
нет — пометить `[NEW-TERM]` (кандидат в `pending`).

**Выход:** показываем 1–3 варианта; **клик вставляет** в ноду (с загрузкой шрифта; оригинал — через
Undo). `intent`+`moment` сохраняются на ноду через `setPluginData`.

## 5. Режим AUDIT
**Вход:** выделение / фрейм / страница. Плагин собирает текст-ноды: `{node_id, layer_name, chars,
coords, соседние лейблы}` + сохранённые `intent`/`moment` (если есть).

**Логика:** отправляем пачкой; LLM возвращает массив находок. Два слоя:
- **rule-based** (`blocker`/`major`/`minor`) — обязателен `rule_id` из свода.
- **эвристика** (`info`, `rule_id:null`, `heuristic:true`) — «не молчать о найденном»: недокументированные
  рассинхроны (авто/автомобиль) + семантика/данные (лейбл≠значение, не сходится сумма). Честно помечать,
  что правила нет.

**Дисциплина:** тишина = валидный результат; не выдумывать замечаний.

**UX:** аудит по **выделению** (как генерация): элемент / секция / экран / несколько экранов.

**Выход в Figma:** сайд-панель со списком находок, **сгруппированных по важности** (blocker/major/minor/info)
с **фильтром** (можно скрыть info/мелкое). `info`-находки визуально отделены (мягкие, «проверьте»).
Действия над находкой:
- **Перейти к ноде** — клик выделяет и проскролливает к тексту.
- **Применить** — заменить текст на `suggestion` в 1 клик (с загрузкой шрифта).
- **Скрыть / игнорировать** — пометить как «не баг», убрать из списка (запоминать на сессию/ноду).
- **Применить всё пачкой** — массовое исправление выбранных находок, **с подтверждением** (не молча).

**Состояния экрана аудита:**
- **Ничего не выделено** — подсказка «Выберите элемент, секцию или экран — и запустите проверку».
- **Идёт проверка** — лоадер «Проверяю тексты…».
- **Чисто (нарушений нет)** ← ключевое пустое состояние. Показать, что проверка РЕАЛЬНО прошла:
  ✓ «Нарушений не найдено» (без «!»); «Проверено N текстов на экране „…"»; честная оговорка
  «Проверка по правилам пройдена — смысл и данные всё равно стоит глянуть глазами» (rule-based ловит
  не всё); действие «Проверить весь экран / страницу» + «Перейти к генерации» (по `EMP-01`).
- **Жёстких нет, есть эвристика** — «Нарушений по правилам нет. Есть N мест на проверку» + список info.
- **Ошибка** — «Не удалось проверить. Проверьте соединение и повторите» (`ERR-01`).

## 6. Схемы вывода (строгий JSON)
**GENERATE:**
```json
{ "variants": [
  { "text": "Оплатить", "rules": ["BTN-01","BTN-05"], "note": "инфинитив, конкретное действие" }
], "new_terms": ["trade-in"] }
```
**AUDIT:**
```json
{ "findings": [
  { "node_id": "123:45", "layer": "button-label", "rule_id": "BTN-02",
    "severity": "major", "quote": "Отправить", "suggestion": "Отправить заявку", "heuristic": false },
  { "node_id": "123:70", "layer": "price", "rule_id": null, "severity": "info",
    "quote": "Итого 227 000 ₸", "suggestion": "Проверьте: сумма строк = 223 000 ₸", "heuristic": true }
] }
```

## 7. Модель контекста и персистентность
- Иерархия: `флоу → экран → элемент`. `intent` задаётся на уровне флоу и наследуется; уже уровень
  переопределяет.
- `intent`/`moment` → `setPluginData` на ноду. Живут в файле. AUDIT переиспользует их, чтобы проверять
  не только форму, но и «в тему ли текст».
- AUDIT без контекста работает (правила формы §4–6 всегда); проверка замысла — только при сохранённом `intent`.

## 8. Приоритет правил при конфликте
Понятность > Терминология (глоссарий) > Тон > Стиль/инфостиль > Типографика.

**Правила vs контекст дизайнера.** Плагин AI-based, но «на рельсах»: свод правил — жёсткий костяк,
контекст/промпт дизайнера — мягкое направление В РАМКАХ правил.
- Корректность (орфография, грамматика), типографика, язык, структура — **правило побеждает**.
  Если запрос дизайнера противоречит (напр. «КАПСом», «на английском») — следуем правилу и **коротко
  поясняем почему** (не молча игнорируем).
- Брендо-настраиваемые пункты (помечены `⚙` — напр. «!»/эмодзи, ё-политика) — контекст дизайнера/бренда
  **может переопределять**.

## 8a. Калибровка экстракции (из реального прогона 189 экранов)
Проверено на практике — иначе много ложных срабатываний:
- **`figma.skipInvisibleInstanceChildren = true`** перед сбором текста: иначе извлекаются СКРЫТЫЕ дефолты компонентов (Subtitle/Caption/Заголовок/Button) и летят как «плейсхолдеры» (в прогоне — 81 ложное из 503).
- **Исключение брендов/бизнес-имён** (LNG-02): «Auto-wash», «Kaspi», «Yandex», названия автомоек/марок — НЕ флагать как «чужой язык». Сверять с глоссарием/списком брендов.
- **Игнорировать chrome устройства** (статусбар «9:41», «Monday…») — не наш текст.

## 9. Открытые тех-решения (перед сборкой)
- ⚙ **LLM-ключ:** clientStorage у дизайнера ИЛИ корп. бэкенд-прокси.
- ⚙ **GLOSSARY_SOURCE:** Google Sheet / Airtable / корп. глоссарий (ждём ссылку).
- ⚙ Оффлайн/лимиты: батчинг AUDIT для больших фреймов, обработка ошибок API.

## 11. Бэклог — добить потом

**Новое (проговорено, но не спроектировано):**
- **Индикация загрузки/прогресса.** Когда плагин работает — чёткое состояние «идёт работа», а не
  ощущение зависания. Спиннер + текст этапа («Собираю тексты…», «Проверяю по правилам…»); для батчей —
  прогресс «обработано X из Y». Обязательно.
- **Контекст всего флоу при генерации одного элемента.** Если дизайнер выбрал одну кнопку — плагин
  должен смотреть на весь флоу/экран (соседние экраны, что до/после, цель шага), а не только на сам
  элемент, чтобы придумать правильный текст. Расширить §7: авто-сбор контекста флоу вокруг выделения.

**UX-мелочи:**
- Первый запуск / настройки: ввод LLM-ключа, подключение `GLOSSARY_SOURCE`.
- Флоу `[NEW-TERM]`: как новый термин из генерации попадает в глоссарий (`pending → approved`).
- **Раздел «Ресурсы / Помощь»** (футер панели или меню «?») со ссылками: гайды в Figma (страница
  «Гайды», по возможности deep-link на конкретные доки — Знаки, Тон, Компоненты и т.д.) и на **глоссарий**
  (когда станет ясно, где он живёт — Q1). Можно добавить и ссылку на `content-guide.md`.

**Ждёт входных данных / решений:**
- Корпоративный глоссарий — ждём ссылку; подключить как живой слой, сверить с решениями.
- Отложенные термины: госномер (C13), ЦОН/спецЦОН (C14), кредит/займ (C21), бренд Mycar (C20/C27).
- C5 «рестайлинг/Рестайл»; C7 экран «Продано» (посыл≠действие); Q1/Q2 глоссарий (где живёт, owner).
- Тех-решения §9: LLM-ключ (clientStorage vs бэкенд), батчинг больших фреймов.

## 10. ПРОМПТ ДЛЯ FIGMA-АГЕНТА (копировать)

```
Собери Figma-плагин «UX Content» для проверки и генерации интерфейсных текстов на русском.

КОНТЕКСТ ПРАВИЛ: приложены файлы content-guide.md и date-time-formats.md — это свод правил.
Забандль их содержимое в плагин как системный контекст для LLM-вызовов. Глоссарий подтягивается
по сети как JSON (схема {canonical, synonyms[], note, status, owner, updated_at}), используем
только status=approved; источник задаётся в настройках (GLOSSARY_SOURCE).

UI: панель с двумя вкладками — «Генерация» и «Аудит». В ОБЕИХ вкладках: свободное поле «Ваш контекст /
промпт» (добавляется к запросу) и кнопка «Прикрепить изображение» (скрин Jira/референс → мультимодальный
запрос, опционально). Работа идёт по ТЕКУЩЕМУ ВЫДЕЛЕНИЮ (элемент/секция/фрейм/экран/компонент); если
ничего не выделено — подсказка «Выберите элемент».

ГЕНЕРАЦИЯ:
- Две кнопки: «Сгенерировать» (с нуля) и «Улучшить» (переписать текст дизайнера).
- Определи тип выделенного элемента, подставь его текст и соседние лейблы. Если в выделении несколько
  текстов — обработай все.
- Поля: moment (дропдаун: рутина/успех/ошибка/онбординг/оплата-безопасность, предзаполнен догадкой),
  intent (текст, наследуется с флоу/экрана), «ваш контекст», constraints (опц.).
- Вызови LLM: system = свод правил; user = собранный контекст. Верни строгий JSON:
  {variants:[{text, rules:[id], note}], new_terms:[]}. 1–3 варианта. Не выдумывай факты →

- Собери текст-ноды выделения/фрейма/страницы: {node_id, layer, chars, coords, соседние лейблы} +

  heuristic}]}. severity: blocker/major/minor/info. Для не-эвристики rule_id обязателен из свода.
  Слой эвристики: rule_id=null, heuristic=true, severity=info — подсвечивай подозрительное, чего нет
  в правилах (рассинхроны терминов, семантика/данные), честно помечая, что правила нет. Тишина —
  валидный результат, не выдумывай замечаний.
- Покажи находки в сайд-панели, сгруппируй по severity + фильтр (скрыть info/мелкое), info визуально

- Состояния: ничего не выделено → подсказка выбрать; идёт проверка → лоадер; ЧИСТО → «Нарушений не
```

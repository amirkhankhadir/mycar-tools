# Помощник по текстам — как пользоваться

Ассистент на базе Claude, который **проверяет и пишет интерфейсные тексты по нашему гайду**. Смысл — единые тексты у всей команды: один свод правил, а не «кто как привык».

---

## Что умеет

- **Проверить** — находит в готовом тексте нарушения гайда (тон, стиль, типографика, термины) и предлагает исправления.
- **Сгенерировать** — пишет текст с нуля под элемент: кнопку, ошибку, пустой экран, заголовок.
- **Улучшить** — переписывает твой черновик по правилам.

Работает **по нашим правилам**, а не «в целом красиво». Если правило конфликтует с твоим запросом — поправит, но коротко объяснит почему.

---

## Как пользоваться

Плагин работает в **Claude Code**. Тексты можно дать двумя способами: вставить в чат или дать ссылку на Figma-фрейм (Claude сам прочитает экран и поймёт контекст).

### Шаг 1. Установить один раз
```
/plugin marketplace add amirkhankhadir/mycar-tools
/plugin install mycar-copy@mycar-tools
```
Обновления: `/plugin marketplace update mycar-tools` → `/plugin update mycar-copy@mycar-tools` (перезапусти сессию).

### Шаг 2. Позвать в нужном режиме
Пиши обычными словами — скилл подхватится сам (можно и `/mycar-copy`). Назови режим: **проверить / улучшить / сгенерировать**.

**Способ А — вставить текст в чат** (быстро):
- «Сделай аудит этих текстов по mycar-copy: …»
- «Улучши этот текст по mycar-copy: …»

**Способ Б — дать ссылку на Figma-фрейм** (плагин читает экран и контекст):
- **Проверить:** «Прочитай тексты из этого Figma-фрейма и сделай аудит по mycar-copy: `<ссылка>`»
- **Улучшить:** «Прочитай тексты из этого фрейма и улучши их по mycar-copy (покажи «было → станет»): `<ссылка>`»
- **Сгенерировать:** «Посмотри этот фрейм, пойми контекст и предложи текст для `<кнопки / заголовка / ошибки…>` по mycar-copy. Момент: `<рутина / успех / ошибка / онбординг / оплата>`. О чём экран: `<фраза>`. `<ссылка>`»

> Для проверки и улучшения хватает ссылки. Для **генерации** добавь строку «момент + о чём экран» — так точнее (иначе плагин переспросит).
> Работа по Figma-ссылке требует, чтобы в сессии был подключён Figma.

### Что получишь
- **Проверка** — список находок: правило (напр. `LNG-03`), важность (blocker / major / minor / info) и «было → станет».
- **Генерация** — 1–3 варианта + какими правилами держатся.
- **Улучшение** — «было → станет» + что изменил и почему.

## Примеры запросов

- «Проверь микрокопию этого экрана по гайду»
- «Напиши заголовок и подпись для пустого состояния каталога»
- «Дай 3 варианта текста кнопки подтверждения заказа»
- «Проверь, нет ли рассинхрона терминов на этих экранах»

---

## Что важно понимать

- **Инструмент на рельсах.** Правила — жёсткий костяк, твой контекст — направление в их рамках. Даёшь больше контекста → точнее результат.
- **Не выдумывает данные.** Если чего-то не хватает — спросит или пометит, а не придумает.
- **Правила живут в одном месте** — репозиторий https://github.com/amirkhankhadir/mycar-tools. Обновили гайд — у всех сразу актуальная версия, ничего переустанавливать не нужно.
- **Нашёл спорное или правила нет?** Инструмент может пометить это как «стоит проверить» (эвристика), даже если явного правила нет. Такие случаи присылай владельцу гайда — так гайд растёт.

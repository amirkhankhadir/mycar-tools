# UX Content — правила интерфейсных текстов + инструмент проверки

Единый источник правил для интерфейсных текстов (RU) и инструмент на его основе: **проверка,
генерация и улучшение** текстов по гайду. Цель — единообразие: один свод правил вместо «кто как привык».

## Две витрины (обе на уже оплаченных местах, без доп. бюджета)

Один источник правды → два инструмента одного формата (спецификация Agent Skills):

- **Claude Code skill** (`/ux-copy`) — для тех, кто работает в Claude Code. Готов и обкатан.
- **Figma skill** — для тех, кто работает в Figma. Собирается из этого же гайда в один `.md` и
  публикуется в Figma на организацию. *(В работе.)*

Плагин Figma — **отложён (фаза 2)**: он требует отдельного API-бюджета, местами Claude Max не
покрывается. Спека сохранена в `plugin-spec.md` на будущее.

## Как пользоваться

См. **`how-to-use.md`** — короткая инструкция для пользователей.

## Как это раздать сотрудникам

См. **`HANDOFF.md`** — что нужно от разработчиков/админов (разместить в корп. репозитории, выдать доступ).

## Состав пакета

**Источник правды (правим здесь):**
- `content-guide.md` — ядро правил (понятность, язык, тон, стиль, типографика, числа, компоненты, глоссарий).
- `date-time-formats.md` — даты/время (`DT-*`, RU & KZ). Ядро §5.1 ссылается сюда.
- *(глоссарий — живой слой, внешний источник; пока не подключён)*

**Инструмент:**
- `.claude/skills/ux-copy/SKILL.md` — Claude Code skill (витрина 1). Ссылается на файлы выше, не копирует их.
- `how-to-use.md` — инструкция для пользователей.
- `HANDOFF.md` — инструкция для разработчиков по размещению и раздаче.

**Для сопровождающих:**
- `audit-eval.md` — золотой тест-корпус (регрессия: прогнать аудит и сверить recall/precision).
- `system-prompt.md`, `plugin-spec.md` — материалы фазы 2 (плагин), отложено.

**Внутреннее (не для пользователей):**
- `open-questions.md` — рабочий журнал обкатки: открытые вопросы, конфликты, решения.

## Как обновлять правила

1. Правишь `content-guide.md` (или `date-time-formats.md`).
2. `git commit` + `git push`.
3. Пользователи Claude Code получают обновление через `git pull`.
4. Для Figma-витрины — пересобрать один `.md` и перепубликовать в Figma.

Правила — единственный источник; обе витрины тянут их, а не хранят свои копии.

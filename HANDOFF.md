# Хендофф разработчикам — разместить и раздать

Просьба к разработчикам одна: **разместить этот репозиторий в корпоративной GitHub-организации
(приватный) и выдать доступ команде.** Дальше подключение и обновление — самообслуживание через
встроенный механизм плагинов Claude Code.

## Что это

Плагин Claude Code (`mycar-copy`) + правила интерфейсных текстов. Технически — набор Markdown-файлов
по спецификации Agent Skills. **Ничего не собирается и не выполняется**, нет зависимостей, секретов,
ключей и внешних вызовов. Работает на **уже оплаченных местах Claude Max** — отдельный API-бюджет
не нужен.

## Что нужно от вас (минимум)

1. Создать **приватный репозиторий** в GitHub-организации компании *(GitLab/Bitbucket — тоже подходят)*.
2. Залить содержимое пакета. Локально уже готов git-репозиторий с коммитами — нужно добавить remote
   и запушить (команды ниже).
3. Выдать **доступ на чтение** нужной команде: дизайнеры и копирайтеры + те, кто ведёт правила.

## Как коллега подключает (ссылка + две команды)

В своём Claude Code:
```
/plugin marketplace add <организация>/<репозиторий>
/plugin install mycar-copy@mycar-tools
```
После этого скилл вызывается командой `/mycar-copy` (или срабатывает по смыслу запроса) в любом проекте.
Клонировать репозиторий и копировать файлы вручную **не нужно**.

## Обновления правил

Правила меняются в `plugins/mycar-copy/skills/mycar-copy/content-guide.md` → `commit` + `push` +
бамп `version` в `plugin.json`. Коллеги получают обновление командой:
```
/plugin marketplace update mycar-tools
```
Для автоматического обновления — включить `autoUpdate` в настройках маркетплейса (`.claude/settings.json`
у пользователя или в корпоративном конфиге).

## Приватный репозиторий — доступ

- **Установка** использует git-credentials пользователя (`gh auth login`, keychain, или SSH-ключ в `ssh-agent`).
- **Фоновое автообновление** приватного репо по умолчанию не использует credentials. Решение — либо
  git-config с токеном:
  ```
  git config --global url."https://x-access-token:<TOKEN>@github.com/<org>/<repo>".insteadOf "https://github.com/<org>/<repo>"
  ```
  либо переменная `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1`.

## Первый пуш (пример GitHub)

```
git remote add origin git@github.com:<организация>/<репозиторий>.git
git push -u origin main
```

## Про Figma

Вторая витрина (Figma-скилл) раздаётся **отдельно** — публикацией файла `figma/mycar-copy.md` в Figma
на организацию. Для неё этот репозиторий пользователям Figma не нужен.

## Чего делать НЕ нужно

Сервер, хостинг, CI/CD, API-ключи, доступ в интернет во время работы скилла.

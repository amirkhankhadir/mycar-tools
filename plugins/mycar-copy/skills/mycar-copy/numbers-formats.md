## 1. Деньги · цена и валюта

- **NUM-02** — Десятичный разделитель — **запятая**: «227 350,50 ₸», не «227 350.50 ₸».
- **LOC-02 (тысячи)** — Разделитель тысяч — **неразрывный пробел**.
  - ✓ `12 000 000 ₸`   ✗ `12000000 ₸`
- **NUM-08** — Знак валюты тенге — **«₸»**, не «тг.» / «т.».
  - ✓ `12 000 000 ₸`   ✗ `12 000 000 тг.`
- **NUM-03** — Между суммой и знаком валюты — **неразрывный пробел**.
  - ✓ `12 000 000 ₸`   ✗ `12 000 000₸`
  - Производные — без точки (house-style `TYP-07`): «₸/мес», не «₸/мес.».
- **NUM-12** — Доллар — знаком **после суммы**, не словом.
  - ✓ `12 000 000 $`   ✗ `12 000 000 долларов`
- **NUM-07** — Тиын: **округляем на витрине** (каталог, карточки, сводки); **точная сумма с тиын — где важна финточность** (оплата, график автокредита, чеки), чтобы сходилось с реальным списанием.
  - ✓ `Оплата: 227 350,50 ₸` (точная сумма на экране оплаты)   ✗ `Оплата: 227 000 ₸` (округлено там, где списывается точная сумма)

## 2. Проценты

- **NUM-05** — В интерфейсе процент пишем **слитно, без пробела**: «5%», «21–23%», «0,1%». С неразрывным пробелом «5 %» — только в научно-техническом/метрологическом тексте (ГОСТ 8.417). Дефолт продукта — интерфейсный (слитно).
- **NUM-09** — Знак **«%»**, не «проц.» / «процентов».
  - ✓ `100%`   ✗ `100 процентов`
- **NUM-10** — Округляем **не больше сотых**.
  - ✓ `99,99%`   ✗ `99,9999999999%`

## 3. Количество

- **LOC-02 (тысячи)** — Разделитель тысяч — неразрывный пробел.
  - ✓ `1 000 000`   ✗ `1000000`
- **NUM-01** — Единицы времени/количества в прозе, таблицах и инпутах — **полным словом**, согласованным с числом («5 дней», «11 опций»); сокращения «дн»/«д»/«ч» — только в date-модуле (относительное время, тесные бейджи).
  - ✓ `5 дней`   ✗ `5 д.`
- **NUM-06** — **Без наращений** (окончаний) к числам.
  - ✓ `3 сентября · первый отзыв`   ✗ `3-е сентября · 1-ый отзыв`

## 4. Единицы измерения

- **TYP-03 / TYP-07** — Между числом и единицей — **неразрывный пробел**; единица **без точки**.
  - ✓ `115 м² · 60 л · 850 000 км`   ✗ `115м² · 60 л.`

## 5. Идентификаторы

- **NUM-11** — Пишем **«ID»** (заглавные), не «id».
  - ✓ `ID заказа`   ✗ `id заказа`

## 6. Дополнения — лучшие практики (вне Figma-витрины)

> Не из витрины `2060:10441`, а из RU-норм (Ководство, Мильчин/Розенталь, редполитика Kolesa, ICU plural)
> и продуктовой специфики. House-style-решения (мощность «л.с.», ИИН сплошным) — приняты командой.

- **NUM-13** — **Суммы показываем полностью**, включая цены в карточках и списках. Цены не сокращаем — пользователь должен видеть точную сумму. Сокращение «1,2 млн ₸» — узкое исключение только там, где место физически не позволяет (напр. подписи осей графика), и по согласованию; по умолчанию — полная сумма.
  - ✓ `1 099 000 ₸` · `1 270 000 ₸` (карточка)   ✗ `1,1 млн ₸` в карточке   ✗ `1099000 ₸`
- **NUM-14** — **В диапазонах и «от/до» единицу/знак валюты не дублируем** — ставим один раз в конце. Тире — по `TYP-02`/`NUM-04`. Основа: Мильчин.
  - ✓ `200 000 – 300 000 ₸`   ✓ `от 200 000 ₸/мес`   ✗ `200 000 ₸ – 300 000 ₸`
- **NUM-15** — **Согласование с числом (plural)** для динамических значений: формы одна/несколько/много. Не «5 день». Для генерируемых чисел выбирать форму по правилу (ICU one/few/many). Основа: RU-грамматика.
  - ✓ `1 день · 2 дня · 5 дней`, `1 объявление · 3 объявления · 10 объявлений`
- **NUM-16** — **В интерфейсе числа — цифрами**, включая 0–9 («3 дня», не «три дня») — для сканируемости и компактности. Словами — только в устойчивых оборотах («в один клик») и когда цифра неудачно открывает предложение. Основа: Ководство, NN/g. (Расширяет `STL-06`.)
- **NUM-17** — **Отсутствие значения — прочерк «—»** (или «нет данных» по контексту), не ложный «0», если ноль вводит в заблуждение. Настоящий ноль показываем цифрой: «Комиссия 0 ₸». Основа: UI best practice.
- **NUM-18** — **Отрицательные и изменения — минус «−»** (U+2212, `TYP-02`): скидка/падение «−15%», «−7 000 ₸» (без пробела); рост «+5%». Не дефис «-». 
- **NUM-19** — **Автомобильные единицы (house-style):** расход «7,5 л/100 км», объём двигателя «1,6 л» / «1 600 см³», пробег «85 000 км», мощность — **«150 л.с.»**. Мощность — осознанное исключение из «без точек» (`TYP-07`): «л.с.» — устоявшаяся в авторынке форма, оставляем с точками для узнаваемости.
- **NUM-20** — **Идентификаторы:** ИИН — **12 цифр сплошным** «990101300123» (как в документах/базах); телефон «+7 700 123 45 67» (группами); госномер как на пластине «123 ABC 02»; VIN — 17 символов заглавными без пробелов (без букв I/O/Q).
